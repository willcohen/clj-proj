PROJ in the Browser

A browser-based interface to PROJ , the cartographic projections and coordinate transformations library, powered by clj-proj. This demo runs the browser build, with a web interface based on wasm-proj by Javier Jimenez Shaw. The same library also targets Node.js and the JVM (native FFI on macOS/Linux/Windows via dtype-next, or GraalWasm fallback on any platform).

How it works

PROJ runs directly in your browser using WebAssembly. The entire library, including its CRS database, runs client-side with no server communication required.

Build:

  • PROJ is compiled to WASM using Emscripten
  • Clojure source is compiled to ES6 modules using Cherry
  • PROJ C function bindings are automatically extracted from fndefs.cljc and exported from WASM
  • Modules are bundled with esbuild

Runtime:

  • PROJ runs in Web Workers, keeping the main thread responsive
  • With Cross-Origin Isolation (COOP/COEP headers), workers use the pthreads WASM build with SharedArrayBuffer; without it, workers fall back to a single-threaded WASM build
  • Each PROJ context is pinned to a specific worker; the pool uses round-robin assignment
  • The proj.db and proj.ini files are fetched at startup and written to Emscripten's virtual filesystem in each worker
  • Grid shift files (NADCON, NTv2, etc.) are fetched on demand from cdn.proj.org when a transformation requires them
  • C-allocated objects are cleaned up using resource-tracker
